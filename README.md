# Camera Coordinate Mapping (top/bottom → door2)

## 📌 Описание задачи

В рамках тестового задания построить модель, которая отображает пиксельные координаты точки с камер `top` или `bottom` в соответствующую точку на изображении основной камеры `door2`.

Формально:

```
(x, y, source) → (x', y')
```

где:

* `(x, y)` — координата на камере `top` или `bottom`
* `source` ∈ {top, bottom}
* `(x', y')` — координата на камере `door2`

---

## 📊 Метрика

Качество оценивается с помощью среднего евклидова расстояния (MED):

```
d = sqrt((x - x')^2 + (y - y')^2)
```

Метрика считается отдельно для:

* `top → door2`
* `bottom → door2`

---

## 🧠 Подход к решению

Решение состоит из двух частей:

---

# 1️⃣ Геометрический baseline (Homography)

### Идея

Для каждой пары кадров:

* есть 17–22 соответствующих точки
* можно восстановить проективное преобразование (гомографию)

Использовалось:

* `cv2.findHomography`
* RANSAC для устойчивости к выбросам

---

### Pipeline

1. Извлечение соответствующих точек
2. Построение матрицы H
3. Применение H к исходным координатам
4. Оценка ошибки

---

### Особенности

* Использовался RANSAC для фильтрации шумных точек
* Дополнительно применялась фильтрация по расстоянию

---

### Результаты

* Raw (без фильтрации): высокая ошибка из-за выбросов
* Filtered: ~1–30 px

---

### Ограничение

Гомография работает **только внутри одной пары кадров**
→ не применима для новых данных напрямую

---

# 2️⃣ ML модель (обобщение)

### Цель

Построить универсальную модель:

```
(x, y, source) → (x', y')
```

которая работает на новых данных

---

## Разработка функционала

Используются следующие признаки:

* Нормализованные координаты:

  * `x / width`
  * `y / height`

* Смещение относительно центра:

  * `dx`, `dy`

* Дополнительные признаки:

  * `r² = dx² + dy²` (радиус)
  * `x * y` (взаимодействие координат)

* Источник камеры:

  * `top = 0`, `bottom = 1`

---

## 🤖 Модель

Используется ансамбль:

* `RandomForestRegressor`

Отдельные модели обучаются для:

* `top`
* `bottom`

Это позволяет учитывать различие геометрии камер.

---

## 📈 Результаты

```
ML MED top: ~120 px
ML MED bottom: ~140 px
```

---

## 🧠 Анализ

* Homography даёт очень высокую точность, но не обобщает
* ML модель хуже по точности, но:

  * работает на новых данных
  * устойчива к шуму
  * не требует парных кадров
